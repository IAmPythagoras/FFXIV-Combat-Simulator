BiS result using algorithm compared to the Balance BiS. All the shown result are according to FFXIVCALC.
All runs were done with only the given player and no 5% team comp bonus.

The rotation executed by the player is given at the start of each sections.

Note that most etro gear set have slightly different mainstat, which results in a slightly off damage/100p value.
However all the sets are compared with the same main stat value of 3378 (except if the gear set uses a 650 item level piece, in which case it is 3368)
and 3338 for tanks.

I realized that some of the rotations wouldn't be possible at high Spell Speed given that the burst phase
is usually at a set time. So it is important to include a couple of 1-2-3 GCD at the end that could buffer the
burst phase.

I have also added the swapDHDetBeforeSpeed option at reaper. This sometime leads to less optimal BiS, so it would be a good idea to try for both value
of this parameter.

It is also important to note that even if faster Speed valued gear set end up having a higher expected DPS, the real important thing
is to figure out how many more actions can you do given a fight. If having higher spell speed does not make you able to do more actions then
it is probably not the best BiS (given no DoT). 

The value of the parameter swapDHDetBeforeSpeed seems to affect (sometime) the final result. However it is not clear if having it set to True
will result in a better or equal to if it was false (and vice versa). I recommend just trying with both value and see the result. At first I assumed this value
should always be set to True since we want the highest DPS before swapping for Speed materias, but it doesn't seem to always agree. Note that having swapDHDetBeforeSpeed
set to True generally makes the solver output higher Speed valued gear set.

The value of oversaturationIterationPost and Pre do have a notable effect at higher speed max value. I recommend having always oversaturationIterationPre set to 1, as so far this has always
resulted in higher DPS gear set for me. But having oversaturationIterationPost to 0 or 1 is a good practice. Having it set to 0 should make Crit less prominent and favor Dh/Det more, but having it set to
1 will most likely favor Crit as it is quite abundant in most gear sets. In my experience, having it set to 0 for high Speed gear sets was the best and it usually didn't have much of an impact for gear sets with
low max speed values. I would still recommend trying it out with different arrangements of such values. However, I have observed that as oversaturationIterationPost and oversaturationIterationPre get higher and higher
the final DPS converges to a value that is usually not the maximum. As explained above, having these values too high will make the solver favor Crit over the others and could result in suboptimal gear sets once you have to
remove all the crit to make it legal.

If swapDHDetBeforeSpeed is not specified it is because both values of True or False resulted in the same gearset.

	Solver BiS 1 might have higher expected DPS over Balance BiS 1 and Solver BiS 2/3, but its higher Spell Speed value will probably be too much for Redmage's rotation and will end up
	wasting mana. Furthermore, the gain in expected DPS of ~6 is too low if you loose mana in the end.
	
	Solver BiS 2 might be one to consider with very similar SPell Speed which could make easier the triple melee combo pot window at the 2 minute mark (pot before burst).
	
	Solver BiS 2 also has ~11 expected more DPS against Balance BiS 1 and finishes 0.23 seconds faster.
	
	It is important to note that 502 Spell Speed might be a better value for oGCD allignment, which the solver cannot figure out (for now).
